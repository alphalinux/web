RedHat 6.0 Problems

This is a list of known problems with RedHat 6.0 for Alpha. This includes the contents of the RedHat errata and other problems that RedHat has yet to fix.

net-tools:

Symptom: Several potential buffer overruns have been corrected within the net-tools package.
Solution: Download and install the following updated package:

  • net-tools-1.52-2.alpha.rpm

talk:

Symptom: The older version 1 talk protocol was broken in Red Hat 6.0.
Solution: Download and install the following updated package:

  • talk-0.11-2.alpha.rpm

mod_php:

Problem: Red Hat Linux 6.0 shipped with PHP 3.0.7. This release of PHP had some problems with glibc 2.1. A later errata release corrected those problems, but it had problems with postgresql, which it intended to support.

This release corrects all known problems with PHP3, as well as adding modular postgresql and imap support.

Solution: Download and install the following updated packages:

  • mod_php3-3.0.9-1.alpha.rpm
  • mod_php3-imap-3.0.9-1.alpha.rpm
  • mod_php3-manual-3.0.9-1.alpha.rpm
  • mod_php3-pgsql-3.0.9-1.alpha.rpm

dev, rxvt, screen:

Security: The /dev/pts filesystem was mounted with options 'mode=0622' in Red Hat Linux 6.0, instead of the correct 'gid=5,mode=0620'. This could lead to users being able to write to affected ttys.

Additionally, once this was corrected, screen and rxvt would still chmod the tty devices to potentially insecure modes.

Solution: Download and install the following updated packages:

  • dev-2.7.7-2.alpha.rpm
  • rxvt-2.6.0-2.alpha.rpm
  • screen-3.7.6-9.alpha.rpm
XFree86:

Symptoms: A number of problems exist with the XFree86 RPMs shipped with Red Hat Linux 6.0. The font server is hardcoded not to accept TCP connections. A race condition leads to slow startups on X servers which are entirely cached in memory. There are some problems with inputting ISO-8859-1 characters with an ISO-8859-2 language in use. By default, the directory /etc/X11/xdm/authdir does not exist, which causes the X server to fall back to no authentication at all.

Additionally, those users who did not use Xkb keyboard extension had problems wwith backspace and Motif applications.

(1999-06-17) The original xinitrc packages released for this update, xinitrc-2.4-1, had errors (the xinitrc and Xclients files were not executable). This has been fixed in the currently available errata pacakges.

Solution: Download and install the following updated packages:

wu-ftpd:

Symptom: file name globbing is broken. 'ftpwho' does not displaying complete information.
Solution: Download the updated wu-ftpd rpm and upgrade the package.

kernel update

Security: Vulnerability allows remote users to crash machines running 2.2.x kernels.
Solution: Depending on if you have a either a single or multi-processor Alpha download:

  • kernel-2.2.5-22.alpha.rpm
  • kernel-smp-2.2.5-22.alpha.rpm

It Is also recommend that you download and install the following.

  • kernel-source-2.2.5-22.alpha.rpm
  • kernel-headers-2.2.5-22.alpha.rpm
  • kernel-doc-2.2.5-22.alpha.rpm
utemper:

Symptom: Several programs on Red Hat 6.0 may not stop running when a user logs out due to a problem handling signals. Programs affect by this bug included xterm, rxvt, and screen.
Solution: Download the updated utemper rpm and upgrade the package.

New boot images:

Symptom: A number of bugs are fixed in this release which include:

  • ICP Vortex RAID controller driver module missing
  • VIA-Rhine ethernet card driver module missing
  • Kickstart installs recrypts passwords even if "--iscrypted" is given
  • Problems obtaining complete DHCP replies from Windows NT DHCP servers (NOTE: for this problem you will also need the new pump package.)
  • Foreign language installs using the hard drive install method
  • Hard drive installs from partitions that will be mounted in the installed system
Solution: Download an updated ramdisk.img

Further Instructions: Copy "ramdisk.img" to a floppy disk using "dd" or "rawrite.exe" then proceed with the instructions in the installation guide.

mod_php3:

Notice:The PHP v3 Apache module shipped with Red Hat Linux 6.0 Web Server 2.0 does not function properly with some PHP code. The PHP team also discovered some issues with PHP and glibc 2.1. For this reason, we are making PHP 3.0.8 available, which corrects these issues. We have also included PostgreSQL and IMAP support on the basis of user demand.
Solution: Download the updated mod_php3 rpm and upgrade the package.

inn:

Security: Security problems have been found with the version of INN that shipped with Red Hat Linux 6.0. By editing the inn.conf file, or changing the INNCONF environment variable, the 'news' user could execute arbitrary code as root.
Solution: Download an and upgrade the following packages:

  • inn-2.2-9.alpha.rpm
  • inn-devel-2.2-9.alpha.rpm
pump:

Symptom: Pump would not allow Linux machines to get an IP from an NT server using DHCP.
Solution: Download an updated pump rpm and upgrade the package.

xscreensaver:

Security: certain security checks were disabled that could result in a security vulnerability if the screensaver was configured in a particular manner. While Red Hat Linux 6.0 does not ship with such a configuration, it is recommended that users upgrade to the fixed packages.
Solution: Download an updated xscreensaver rpm and upgrade the package.
